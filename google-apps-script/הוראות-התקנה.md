# תיקון שגיאה: Bad value ב-include

## מה השגיאה אומרת?
ב-Apps Script **חסרים קבצי HTML** בשמות הנכונים.  
`include('Styles')` מחפש קובץ בשם **`Styles`** – לא `Styles.html` ולא קוד בתוך `Code.gs`.

---

## פתרון הכי פשוט (קובץ אחד בלבד)

### 1. ב-Apps Script מחק את כל קבצי ה-HTML הישנים (אם יש)

### 2. צור קובץ HTML חדש אחד
- לחץ **+** → **HTML**
- שם הקובץ: **`Index`** (בדיוק כך, באנגלית)

### 3. העתק תוכן
פתח במחשב:
`google-apps-script/Index-להדבקה-ב-AppsScript.html`  
העתק **הכל** → הדבק בעורך ה-`Index` ב-Apps Script → שמור

### 4. Code.gs
רק קובץ **`Code.gs`** (סקריפט) – העתק מ-`Code.gs` בפרויקט

### 5. בדיקה
- בחר פונקציה: **`testListContacts`** → הרץ → Logs → `נמצאו XXX אנשי קשר`
- בחר: **`testHtmlFiles`** → הרץ → Logs → `OK: Index`

### 6. פריסה (חובה גרסה חדשה!)
Deploy → **Manage deployments** → ✏️ → **New version** → Deploy  
(בלי גרסה חדשה השינויים לא יופיעו באתר)

### 7. אם עדיין ריק
- ודא ש-`Code.gs` כולל `getBootstrapContactsJson_`
- ודא שב-`Index` יש שורה: `<script type="application/json" id="contacts-bootstrap">`
- פתח את ה-URL של ה-Web App (לא תצוגה מקדימה בעורך)

### 8. אם הלוגו לא מופיע (אייקון שבור בכותרת)

הלוגו נטען מ־`brand-logo-bootstrap` (JSON) ב-JavaScript – **לא** מ־`src` בתבנית (נשבר ב-Apps Script).

1. במחשב: `node scripts/build-index.mjs`
2. העתק **`Code.gs`** + **`Index-להדבקה-ב-AppsScript.html`** ל-Apps Script.
3. ודא ב־`Index`: `id="brand-logo-bootstrap"` ו־`data-logo-v="15"`.
4. **פריסה:** Deploy → **New version** → Deploy.
5. פתח **`/exec`** (לא `/dev`).
6. הרץ `testLogoSetup` – אמור להופיע `bootstrap JSON: {"type":"base64"...`.
7. אם `LOGO_URL` ב-Properties שגוי – מחק אותו.

### 9. אייקון מסך הבית (Android) – לא אייקון G

ב-**Apps Script** תגיות `<link rel="icon">` ב-HTML **לא משפיעות** – רק `setFaviconUrl()` ב-`doGet` (כבר ב-`Code.gs`).  
נדרש PNG ב-**Drive** (שיתוף ציבורי); `setFaviconUrl` משתמש ב-`https://drive.google.com/uc?id=...&export=download&format=png` (חובה `format=png`).

**א. אוטומטי:**

1. העתק `Code.gs` + `Index-להדבקה-ב-AppsScript.html` + `appsscript.json`
2. הרץ `authorizeScript` (או `setupPwaIconOnDrive`) → אשר Drive
3. הרץ `refreshPwaIconUrls` – בלוג חייב להופיע: `lh3.googleusercontent.com/d/...`
4. **בטלפון (ללא התחברות):** פתח את הקישור מהלוג – חייב להופיע לוגו PNG
5. Deploy → **New version** → Deploy
6. מחק **כל** קיצורי "אנשי קשר" מהמסך הבית
7. פתח `/exec`, המתן ללוגו בכותרת, ⋮ → **התקן אפליקציה**

**ב. ידני (אם `createFile` נחסם):**

1. העלה `logo-yeshiva.png` ל-Drive → שיתוף **כל מי שיש לו את הקישור**
2. `registerPwaIconDriveFile('מזהה-או-קישור')` → `refreshPwaIconUrls`
3. New version → מחק קיצור → הוסף מחדש

**בדיקה:** `?action=manifest` – ב-`icons[].src` חייב `googleusercontent.com` (לא `script.google.com`).

### 10. למה עדיין רק «קיצור דרך» ולא «התקן אפליקציה»?

זו **מגבלת Chrome** על `script.google.com` – אי אפשר PWA מותקנת משם.

**פתרון:** פרוס את `pwa-host/` (אחרי `node scripts/build-pwa-host.mjs`) ל-GitHub Pages / Firebase, וחבר ל-API של Apps Script דרך `config.js`.

מדריך: [`../pwa-host/README.md`](../pwa-host/README.md)

## פתרון חלופי (3 קבצי HTML)

| שם ב-Apps Script | קובץ במחשב |
|------------------|-------------|
| `Index` | Index.html (בלי שורות include – ראה למטה) |
| `Styles` | Styles.html |
| `Client` | Client.html |

ב-Index.html ב-Apps Script חייבות להישאר השורות:
```html
<style><?!= include('Styles'); ?></style>
...
<script><?!= includeScript('Client'); ?></script>
```

---

## מבנה סופי ב-Apps Script

```
📁 פרויקט Apps Script
├── Code.gs          ← סקריפט
└── Index.html       ← HTML (שם: Index)
```

**אין** צורך ב-Styles ו-Client אם השתמשת בקובץ המאוחד.
